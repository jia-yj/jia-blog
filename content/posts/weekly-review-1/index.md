## 三、本周关键决策

**1. Hugo 而非 Astro**

- 背景：需要搭建以摄影为主的个人博客
- 结论：选 Hugo
- 推理：已有成熟主题（PaperMod），静态生成快，Vercel 部署简单；Astro 更适合需要交互组件的场景

**2. Cloudflare R2 存储图片**

- 背景：图片体积大（单张最高 11MB），不适合放 Git 仓库
- 结论：接入 R2，通过 Public URL 在 Markdown 中引用
- 推理：免费额度大，CDN 自带，和 Vercel 配合无缝

**3. watch 脚本而非单次脚本**

- 背景：PDF 处理是频繁操作
- 结论：用 `mineru_watch.py` 常驻监控
- 推理：放入 pdf_inbox 即可自动处理，减少手动触发摩擦

**4. 用 CLAUDE.md 而非每次对话来传递学术研究规则**

- 背景：每次开新会话都要重新解释"写论文要怎么做"，效率低
- 结论：把完整的研究/阅读/写作方法论写入 `write/CLAUDE.md`，进入该目录自动生效
- 推理：规则是稳定的（学术方法论不会每周变），适合放进持久指令；具体任务才需要每次对话
